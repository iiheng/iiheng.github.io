docker换镜像源以解决拉取镜像速度慢的问题

docker

前言

docker 换镜像源以解决拉取镜像速度慢的问题

  1. 想必大家在安装完docker，兴高采烈的就去拉取自己想要的镜像时，会发现拉取的速度不仅很慢抑或是被中断，就像下面这样：
    read tcp xxx:xxx->xxx:xxx: read: connection reset by peer

    当然也可能会出现下面这个问题

    Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running

    这些问题都是可以通过修改镜像源来解决的

  1. 何为镜像源呢，就是我们拉取镜像的地方，刚下载的docker，它的镜像源位于国外，并且国内也是限制的比较严，所以会很慢,所以我们要做的只需将获取镜像的地址换成国内的就可以了，以下是国内几个常用的镜像源。（推荐使用阿里云的镜像源，镜像源地址需要自己登录获取，不要用下面那个，其他步骤一样，链接在下面阿里云镜像源那里，点进去，然后登录就可以获取了）

镜像源

Docker中国区官方镜像:

https://registry.docker-cn.com

网易：

http://hub-mirror.c.163.com

中国科技大学：

https://docker.mirrors.ustc.edu.cn

阿里云：

阿里云具体看官方（很简单，详细）：阿里云容器镜像服务

https://cr.console.aliyun.com/

修改（替换）镜像源

进入/etc/docker/目录

cd /etc/docker

没有就创建：

sudo mkdir -p /etc/docker

查看是否存在daemon.json文件

ls

certs.d daemon.json key.json seccomp.json

修改daemon.json文件（不存在会自动创建）

vim daomon.json

按i键进行输入

{
  "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
}

配置多个镜像源

{
  "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn",
		       "https://registry.docker-cn.com",
  		       "http://hub-mirror.c.163.com" ]
}

按esc左上角的退出键进入命令模式
然后再输入:wq保存退出，注意:
然后我们再用ls命令看看有没有了

重新加载daemon和docker

输入以下命令就可以重新加载和重启了

sudo systemctl daemon-reload 
sudo systemctl restart docker

测试

docker pull germey/proxypool

哈哈，比之前快多了。ctrl+C中止拉取镜像
